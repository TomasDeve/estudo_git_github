1 - INIT

- git init (Para iniciar o git na pasta específica)

2 - CLONE

- git clone "Link do repo"  (Para baixar um repositório) (feito 1 vez só)

3 - ADD

- git add . (adicionar todas as mudanças)

4 - COMMIT 

- git commit "nome do arquivo" -m "mensagem" (Commit de um arquivo específico)
- git commit -a -m "menssagem" (-a vem de all, commit de tudo)

5 - PUSH 

- git status está ok!
- git push (envia tudo para o repo)

6 - PULL

- git pull (comando utilizado para baixar as alterações feitas de outros locais no github)

7 - RM 

git rm "nome do arquivo"

8 - LOG

- git log (Ver o histórico de commits)

============================

DESFAZENDO ALTERAÇÕES 

1 - CHECKOUT 

EXPLICAÇÃO: A pasta volta ao estado do último push

- git checkout "nome da pasta ou arquivo"

=================================

GIT IGNORE 

EXPLICAÇÃO: Serve para ignorar alguns arquivos ou pastas na hora de mexer com git (fazer commits ou push)

1 - Criar pasta ".gitignore" no repositório principal. 
2 - dentro da pasta posso colocar o caminho do arquivo a ser ignorado em cada linha.
